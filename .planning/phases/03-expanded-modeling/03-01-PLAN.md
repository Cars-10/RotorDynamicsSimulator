# Plan: Material System Implementation

## Objective
Replace the current color-based material placeholder with a physics-ready material system including density and Young's modulus.

## Execution Context
- **Source**: `src/constants/materials.ts` (New), `src/types.ts`, `src/components/ShaftEditor.tsx`
- **Goal**: Enable physics-based modeling.

## Context
Phase 3 requires physical properties for accurate modeling. Currently, `ShaftSegment` only stores a `color`. We need to introduce a `Material` entity and link segments to it.

## Tasks
- [ ] **Define Material Types**: Create `Material` interface and a registry of standard materials (Steel, Aluminum, Titanium) in `src/constants/materials.ts`.
- [ ] **Update Data Model**: Modify `ShaftSegment` in `src/types.ts` to include `materialId` and remove explicit `color` (or make color derived).
- [ ] **Migration Logic**: Update `useSimulation.ts` to handle initialization and potential migration of old data (defaulting to Steel).
- [ ] **UI Update**: Refactor `ShaftEditor.tsx` to allow selecting a material from a list instead of cycling colors. Display material properties (E, rho) in a tooltip.
- [ ] **Verification**: Ensure segments render with the correct color associated with the selected material.

## Success Criteria
- `ShaftSegment` has a `materialId`.
- Users can select "Steel", "Aluminum", etc., from the UI.
- The visualizer colors update to match the material.
- Types are strict.

## Output
- Modified `src/types.ts`
- New `src/constants/materials.ts`
- Modified `src/components/ShaftEditor.tsx`
