# Plan: Advanced Component Data Models

## Objective
Expand `RotorComponent` to support detailed physical properties for bearings and seals.

## Execution Context
- **Source**: `src/types.ts`, `src/hooks/useSimulation.ts`
- **Goal**: Prepare data structures for MOD-01 and MOD-02.

## Context
Bearings and seals are currently just markers with a type. Advanced analysis (Phase 4) requires stiffness and damping coefficients.

## Tasks
- [ ] **Define Physics Interfaces**: Add `SpeedDependentCoefficients` (supporting constant or polynomial values) and `StiffnessDampingMatrix` to `src/types.ts`.
- [ ] **Update RotorComponent**: Add optional `physics` field to `RotorComponent` (specifically for Bearings/Seals) using these new types.
- [ ] **Data Initialization**: Update `useSimulation.ts` to provide default constant values (flat speed dependence) when a new bearing/seal is added.
- [ ] **Type Guard Utils**: Create helper functions (e.g., `isBearing(comp)`) in `src/utils/physicsUtils.ts` to safely access these properties.

## Success Criteria
- `types.ts` reflects the physics requirements (Kxx, Kyy, Cxx, Cyy).
- New components instantiate with valid default physics data.
- No UI changes yet (just data layer).

## Output
- Modified `src/types.ts`
- Modified `src/hooks/useSimulation.ts`
