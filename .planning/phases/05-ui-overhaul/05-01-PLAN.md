# Plan: UI Overhaul & Layout Restructuring

## Objective
Complete visual transformation of the application into a modern, professional-grade engineering tool with a specific "Dark Blue" theme and a multi-viewport layout optimized for rotor dynamics analysis.

## Execution Context
- **Phase**: 5 (UI Overhaul)
- **Source**: .planning/phases/05-ui-overhaul/05-01-PLAN.md
- **Dependency**: Phase 4 (Analysis)

## Context
The current UI is functional but lacks professional polish ("really ugly"). The goal is to implement a specific layout:
1.  **Radial View**: Top-Left region.
2.  **Isometric View**: Center background, oriented diagonally (Bottom-Left to Top-Right).
3.  **Longitudinal View**: Bottom-Right region.
4.  **Theme**: Deep Dark Blue background, modern "engineering" aesthetic.

## Tasks

### 1. Theme Modernization
- [ ] **Update Tailwind Config**: 
    - Change `canvas` color to a deep blue (e.g., Slate-950 `#020617` or similar).
    - Update `panel`, `surface`, and `border` colors to compatible blue-grey tones.
    - Ensure `primary` (Cyan) pops against the new background.
- [ ] **Global Styles**: Update `src/index.css` to remove any conflicting hardcoded styles and ensure the body background matches the theme.

### 2. Layout Restructuring
- [ ] **Refine VisualizerContainer**: 
    - Modify `src/components/visualizer/VisualizerContainer.tsx`.
    - Ensure `ViewType.ALL` is the primary mode.
    - Position `RadialView` strictly in the top-left.
    - Position `LongitudinalView` strictly in the bottom-right.
    - Adjust `IsometricView` (if needed) to ensure the rotor runs diagonally from bottom-left to top-right (may require camera tweaking in `IsometricView.tsx` or passing specific props).

### 3. Component Styling Updates
- [ ] **Update Header**: Ensure `src/components/layout/Header.tsx` blends with the new dark blue theme (glassmorphism or solid dark blue).
- [ ] **Update Sidebar**: Style `src/components/layout/Sidebar.tsx` to match the new `panel` colors.
- [ ] **Update Controls**: Ensure `ViewportControls` and buttons look modern and "flat" or "subtle" to not distract from the 3D view.

### 4. Verification
- [ ] **Visual Check**: Verify the "Dark Blue" theme is applied globally.
- [ ] **Layout Check**: Confirm Radial (TL), Iso (Center/Diag), and Long (BR) positioning.
- [ ] **Responsiveness**: Ensure the layout doesn't break completely on smaller screens (though target is likely desktop).

## Output
- Updated `tailwind.config.js`
- Updated `VisualizerContainer.tsx`
- Refined UI components
